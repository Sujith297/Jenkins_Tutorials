# Jenkins Declarative Pipeline with Maven

In this tutorial, we will create a Declarative Pipeline that performs the following:

* Checks out the source code.
* Builds the application using Maven.
* Archives the generated artifacts.
* Waits for manual approval before deployment.
* Deploys the application.
* Sends an email notification after every pipeline execution.

## Prerequisites

* Jenkins installed and configured.
* Maven configured in **Manage Jenkins → Tools** with the name `mymaven`.
* An agent node configured with the label `dev`.
* Email Notification configured in Jenkins.

## Generating Pipeline Syntax

Jenkins can automatically generate Pipeline syntax for most steps.

1. Open your Pipeline job.
2. Go to **Configure**.
3. Scroll down to the **Pipeline** section.
4. Click **Pipeline Syntax**.
5. Select the desired step (for example, `git`, `archiveArtifacts`, `mail`, `input`, etc.).
6. Enter the required values.
7. Click **Generate Pipeline Script**.
8. Copy the generated syntax and paste it into your `Jenkinsfile`.

> Most of the Pipeline syntax can be generated by configuring the same options that are available in a Freestyle job.

## Pipeline Script

```groovy
pipeline {

    agent {
        label 'dev'
    }

    tools {
        maven 'mymaven'
    }

    stages {

        stage('Code') {
            steps {
                git branch: 'main',
                url: 'https://github.com/your-username/your-repository.git'
            }
        }

        stage('Build') {
            steps {
                sh 'mvn clean install'
            }
        }

        stage('Artifact') {
            steps {
                archiveArtifacts artifacts: 'target/*.war', fingerprint: true
            }
        }

        stage('Deploy') {
            input {
                message "Ready to deploy?"
                ok "Deploy"
            }

            steps {
                echo 'Deploying application...'
            }
        }
    }

    post {

        always {

            mail(
                to: 'example@gmail.com',
                subject: 'Jenkins Pipeline Build Notification',
                body: """
Job Name : ${env.JOB_NAME}
Build Number : ${env.BUILD_NUMBER}
Build Status : ${currentBuild.currentResult}

Build URL : ${env.BUILD_URL}
"""
            )
        }
    }
}
```

## Configuring Email Notification

1. Go to **Manage Jenkins**.
2. Select **System**.
3. Navigate to the **E-mail Notification** section.
4. Configure the following:

```text
SMTP Server : smtp.gmail.com
SMTP Port   : 465
Use SSL     : Enabled
```

5. Provide:

* Sender Email Address
* Authentication Username (Gmail)
* Authentication Password (Gmail App Password)

6. Click **Test Configuration** and verify that the email is received successfully.
